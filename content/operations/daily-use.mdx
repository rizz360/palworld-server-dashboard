# Daily Use

## Start of session checklist

When opening the dashboard, check:

- connection state is `connected`
- server name and version look correct
- player count matches expectation
- FPS and frame time are reasonable
- FPS history is populated

## Before maintenance

Recommended flow:

1. Send an announcement with the planned maintenance time.
2. Save the world.
3. Warn players again shortly before the action.
4. Shut down or restart through your configured process.
5. Confirm the server returns and the dashboard reconnects.

## Moderation workflow

For player incidents:

1. Locate the player in the roster.
2. Confirm identity using name, user ID, and context.
3. Use kick for temporary removal.
4. Use ban for severe or repeated issues.
5. Keep a separate moderation log if your community needs audit history.
