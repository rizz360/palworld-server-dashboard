# Monitoring

The monitoring panels help operators understand server health at a glance.

## Live metrics

The dashboard displays Palworld metrics such as:

- server FPS
- frame time
- current players
- maximum players
- uptime
- world day
- base count

## FPS health summary

The FPS panel summarizes performance using several signals instead of a single point-in-time value:

- current FPS
- minimum FPS in the window
- average FPS
- maximum FPS
- median FPS
- longest continuous dip below 45 FPS
- percentage of the window below 30 FPS

The health verdict avoids pretending there is certainty when the sampler has no data, stale data, or too little data.

The sampler runs inside the dashboard process and is enabled with `PALWORLD_FPS_SAMPLER=true`. When it is not enabled, nothing maintains the history ring and these signals stay empty.

## Snapshot polling

Metrics and players are refreshed through `/api/server-snapshot`. This reduces request count and keeps panels in sync.

## Missing data

A blank or stale metric usually means:

- the Palworld REST API is unavailable
- the server is restarting
- credentials are wrong
- the in-process FPS sampler is not enabled (no `[fps-sampler]` activity in the dashboard logs)
- the history file is unreadable
