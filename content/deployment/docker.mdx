# Docker Deployment

## Published image

The default Compose file uses:

```text
ghcr.io/rnz01/palworld-server-dashboard:latest
```

Override with:

```env
PALWORLD_SERVER_DASHBOARD_IMAGE=ghcr.io/rnz01/palworld-server-dashboard:sha-...
```

## Ports

The container listens on port `3000`. Change the host port with:

```env
PALWORLD_SERVER_DASHBOARD_PORT=3001
```

Then open:

```text
http://localhost:3001
```

## Persistent volumes

Compose defines:

| Volume | Purpose |
| --- | --- |
| `palworld-dashboard-data` | Panel credential hashes. |
| `palworld-fps-metrics` | FPS history ring. |

Do not delete `palworld-dashboard-data` unless you intentionally want to reseed panel passwords.

## Recreate cleanly

```bash
docker compose up -d --force-recreate
```

## Remove stack

```bash
docker compose down
```

This keeps named volumes. To delete volumes too:

```bash
docker compose down -v
```

Only use `-v` if you understand that it removes dashboard credential state and FPS history.
