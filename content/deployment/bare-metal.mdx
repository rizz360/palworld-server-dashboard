# Bare Metal

Bare-metal deployment runs the Next.js standalone server directly on a host.

## Build

```bash
npm ci
npm run build
```

## Start

```bash
npm run start
```

By default the app listens on port `3000`.

## Environment

Store production environment variables in a file readable by the service manager, for example:

```text
/etc/palworld-dashboard/dashboard.env
```

Keep it mode `600` because it contains secrets.

## systemd shape

A minimal service should:

- run as an unprivileged user
- set `NODE_ENV=production`
- load the dashboard environment file
- restart on failure
- write `PANEL_AUTH_FILE` to a persistent writable path

If the unit is hardened (for example with `ProtectSystem=strict`), it also needs write access to any directory the dashboard writes to. Grant it through `ReadWritePaths=`, including the FPS ring file's directory (`PALWORLD_FPS_HISTORY_FILE`, default `./data/`) when the sampler is enabled.

## FPS sampler

The FPS sampler runs inside the dashboard server itself — there is no separate process or unit to install. Set `PALWORLD_FPS_SAMPLER=true` in the dashboard environment file to enable it. The sampler reuses the dashboard's REST configuration and writes rolling history to `PALWORLD_FPS_HISTORY_FILE` (default `./data/fps-history.json`), which the panel reads back. See [FPS History](/docs/configuration/fps-history) for details.

## Journal chat

If using the live chat reader, add the dashboard user to `systemd-journal` or otherwise grant read-only journal access.
