# Security

Palworld Server Dashboard controls privileged server actions. Deploy it like an internal admin tool.

## Security boundaries

The dashboard includes several important protections:

- Browser cannot choose the upstream REST host.
- Real Palworld admin password is injected server-side only.
- Panel passwords are hashed with scrypt in `PANEL_AUTH_FILE`.
- Moderator tier is enforced server-side with an endpoint allowlist.
- Invalid password attempts are rate limited.
- Passwords are sent in headers, not query strings.

## Your responsibilities

You are responsible for:

- protecting the dashboard URL
- keeping `.env` private
- using strong unique passwords
- securing the host running the panel
- backing up world saves and important config
- reviewing updates before production use

## Recommended deployment model

Best:

```text
Trusted operators → VPN/private network → dashboard → Palworld REST API
```

Acceptable with care:

```text
Trusted operators → HTTPS reverse proxy with auth → dashboard → Palworld REST API
```

Avoid:

```text
Public internet → dashboard with only panel password → Palworld REST API
```

## Public read-only view

The optional `/view` page is the one surface designed for public exposure. It is disabled by default (`PUBLIC_VIEW_ENABLED`), requires no credentials, performs no actions, and serves only an allowlisted snapshot: metrics plus player name, level, and map position — never IPs, Steam/user IDs, ping, settings, or passwords. Its snapshot is cached server-side, so public traffic cannot load the game server.

If you publish it, expose only `/view` and `/api/public-view` (plus static assets) at your reverse proxy and keep every other route restricted. Remember that player names and live positions become public — set `PUBLIC_VIEW_ANONYMIZE_NAMES=true` to replace names with stable pseudonyms, or see [Public Read-Only View](/docs/features/public-view).

## Browser storage

If remember-me is enabled, the browser stores panel login data locally. This is convenient but means anyone with access to that browser profile may be able to use the dashboard.

Do not use remember-me on shared devices.

## Moderator tier caveat

The mod tier limits dashboard API access, but moderators still perform real actions such as kick and ban. Treat mod passwords as privileged credentials.

## Incident response

If a credential leaks:

1. Rotate the panel admin/mod password from settings or reseed `PANEL_AUTH_FILE`.
2. Rotate the real Palworld admin password.
3. Restart the dashboard and Palworld server if needed.
4. Review reverse proxy and dashboard logs.
