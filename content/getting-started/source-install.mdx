# Source Install

Use a source install when you are developing the dashboard or deploying it without Docker.

## 1. Clone and install

```bash
git clone https://github.com/RNZ01/palworld-server-dashboard.git
cd palworld-server-dashboard
npm install
```

## 2. Configure environment

```bash
cp .env.example .env
```

Set at minimum:

```env
PANEL_INITIAL_ADMIN_PASSWORD=replace-with-a-panel-password
PALWORLD_ADMIN_PASSWORD=replace-with-real-game-admin-password
PALWORLD_REST_URL=http://127.0.0.1:8212
```

Use the address that is reachable from the machine running the dashboard.

## 3. Run for development

```bash
npm run dev
```

Open:

```text
http://localhost:3000
```

## 4. Run for production

```bash
npm run build
npm run start
```

The build also creates the Pagefind search index used by the documentation.

## FPS sampler

The FPS sampler runs inside the dashboard server itself — there is no separate process to start. Enable it by setting `PALWORLD_FPS_SAMPLER=true` so rolling FPS history is collected even while nobody has the dashboard open:

```env
PALWORLD_FPS_SAMPLER=true
PALWORLD_FPS_HISTORY_FILE=./data/fps-history.json
```

The sampler reuses the dashboard's `PALWORLD_REST_URL` and `PALWORLD_ADMIN_PASSWORD`. A single `PALWORLD_FPS_HISTORY_FILE` variable is both written by the sampler and read by the panel; it defaults to `./data/fps-history.json`. Make sure that directory is writable by the dashboard process.
