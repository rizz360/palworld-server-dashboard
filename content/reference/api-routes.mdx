# API Routes

The dashboard uses internal Next.js API routes. These are implementation details, but understanding them helps with troubleshooting.

| Route | Purpose |
| --- | --- |
| `/api/auth-tier` | Resolves a submitted panel password to admin, mod, or invalid. |
| `/api/palworld/[...path]` | Authenticated proxy to Palworld REST endpoints. |
| `/api/server-snapshot` | Combined metrics, players, and FPS history request. |
| `/api/fps-history` | Reads the server-side FPS ring. |
| `/api/chat` | Reads recent chat/join/leave events from the server log (a log file, `docker logs`, or journald). |
| `/api/server-restart` | Writes optional host restart request/cancel files. |
| `/api/panel-auth/admin-password` | Admin password management endpoint. |
| `/api/panel-auth/mod-password` | Moderator password management endpoint. |
| `/api/demo-mode` | Reports whether runtime demo mode is enabled. |
| `/api/public-view` | Unauthenticated, allowlist-sanitized snapshot for the optional public `/view` page. 404 unless `PUBLIC_VIEW_ENABLED=true`. |

## Authentication

Protected routes expect the panel password in the internal dashboard header, not in query strings.

`/api/public-view` is the deliberate exception: it is opt-in, read-only, accepts no client input, and serves only an allowlisted payload (no IPs, IDs, ping, settings, or history). See [Public Read-Only View](/docs/features/public-view).

## Mod tier enforcement

The Palworld proxy checks the resolved password tier on each request. Mod users are limited to an exact method/path allowlist.
